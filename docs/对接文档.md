# 表单对接文档

本文档说明流程设计器「表单」能力的对接方式，包含：

- 何时展示表单面板、如何用 slot 替换默认表单面板
- 表单 ID（表单标识）的存储逻辑
- 表单对象（FormData/FormField/Validation/Properties/Value）的数据结构与 BPMN XML 映射
- 自定义实现时如何安全地读写 BPMN 元素属性与扩展元素

## 1. 表单面板的展示范围

属性面板在以下元素类型上展示「表单」折叠项：

- `StartEvent`
- `UserTask`

对应逻辑在 [PropertiesPanel.vue](../package/penal/PropertiesPanel.vue#L201)。

## 2. 表单面板的两种接入方式

### 2.1 默认接入（ElementForm）

未传入 `form` slot 时，属性面板渲染默认组件 [ElementForm.vue](../package/penal/form/ElementForm.vue)：

- 「表单标识」：写入/读取 `formKey`
- 「业务标识」：写入/读取 `FormData.businessKey`
- 「表单字段」：维护 `FormData.fields`（字段列表），支持枚举值、校验约束、字段属性

### 2.2 自定义接入（推荐：form slot 覆盖默认面板）

属性面板支持通过 `form` slot 完全替换默认的 `<element-form />` 渲染：

- 传入 `form` slot：只渲染 slot 内容，默认表单面板不渲染
- 未传入 `form` slot：渲染默认 `<element-form />`

实现位置：[PropertiesPanel.vue](../package/penal/PropertiesPanel.vue#L16-L22)

slot 暴露参数：

```ts
{
  id: string,                               // 当前元素 ID
  formKey: string,                          // 当前表单标识
  setFormKey: (formKey: string) => void,    // 写回 formKey
  values: Record<string, string>,           // 业务扩展键值（与 ElementBusinessConfig 的 values 一致）
  emit: (name: string, value: string) => void // 写回业务扩展（与 ElementBusinessConfig 的 emit 一致）
}
```

父组件用法示例：

```vue
<MyPropertiesPanel :bpmn-modeler="modeler">
  <template v-slot:form="{ id, formKey, setFormKey, values, emit }">
    <MyCustomFormPanel :element-id="id" :form-key="formKey" @update:form-key="setFormKey" :business="values" @update:business="emit" />
  </template>
</MyPropertiesPanel>
```

### 2.3 通过封装组件透传 slot（重要）

如果你在自己的封装组件中使用了 `<my-properties-panel>`，并希望把上层的 `form` slot 透传进去，需要注意：

- 不要无条件声明 `v-slot:form`，否则 `<my-properties-panel>` 会认为“已传入 form slot”，从而隐藏默认 `<element-form />`，即使你上层并没有提供任何内容
- 推荐做法：只有当封装组件自身收到了 `form` slot 时，才把它传给 `<my-properties-panel>`

示例（Vue2）：

```vue
<my-properties-panel ...>
  <template v-if="$scopedSlots.form" v-slot:form="data">
    <slot name="form" v-bind="data" />
  </template>
</my-properties-panel>
```

## 3. 表单 ID（表单标识）存储逻辑

### 3.1 存储位置

「表单标识」在 BPMN 元素的 `businessObject.formKey` 上存储：

- 读取：`this.formKey = this.bpmnElement.businessObject.formKey;`
- 写入：`modeling.updateProperties(this.bpmnElement, { formKey: this.formKey })`

对应代码见 [ElementForm.vue](../package/penal/form/ElementForm.vue#L196-L221)。

### 3.2 XML 映射（与 prefix 关系）

属性面板通过 `prefix`（默认 `camunda`）决定引擎扩展命名空间：

- `prefix = "camunda"`：XML 中通常表现为 `camunda:formKey="xxx"`
- `prefix = "activiti"`：XML 中通常表现为 `activiti:formKey="xxx"`

`formKey` 的可序列化能力来自对应引擎 descriptor 中 `FormSupported.formKey` 的声明（`isAttr: true`）：

- Camunda： [camundaDescriptor.json](../package/designer/plugins/descriptor/camundaDescriptor.json#L198-L215)
- Activiti： [activitiDescriptor.json](../package/designer/plugins/descriptor/activitiDescriptor.json#L197-L214)

示例（以 camunda 为例）：

```xml
<bpmn:userTask id="UserTask_1" camunda:formKey="FORM_001">
  ...
</bpmn:userTask>
```

## 4. 表单对象设计（FormData / FormField 等）

除 `formKey` 外，字段级配置存储在 `bpmn:extensionElements` 下的引擎扩展元素中。

### 4.1 根对象：FormData

存储位置：`bpmnElement.businessObject.extensionElements.values` 中的 `${prefix}:FormData`

- 类型：`${prefix}:FormData`（如 `camunda:FormData`）
- 字段：
  - `businessKey?: string`（属性，isAttr）
  - `fields: FormField[]`（子元素数组）

读/建逻辑：

- 若元素没有 `extensionElements`，会创建：`bpmn:ExtensionElements({ values: [] })`
- 若找不到 `${prefix}:FormData`，会创建：`${prefix}:FormData({ fields: [] })`
- 之后会调用 `updateElementExtensions()` 写回，确保扩展元素存在，避免后续操作报错

对应代码见 [ElementForm.vue](../package/penal/form/ElementForm.vue#L196-L218)。

XML 示例：

```xml
<bpmn:userTask id="UserTask_1" camunda:formKey="FORM_001">
  <bpmn:extensionElements>
    <camunda:formData businessKey="applyNo">
      ...
    </camunda:formData>
  </bpmn:extensionElements>
</bpmn:userTask>
```

### 4.2 字段对象：FormField

类型：`${prefix}:FormField`（如 `camunda:FormField`）

基础字段：

- `id: string`（字段 ID）
- `label?: string`（字段名称）
- `type: string`（字段类型：`string | long | boolean | date | enum | custom`）
- `defaultValue?: string`
- `datePattern?: string`（当 `type = date` 时可用）

扩展字段：

- `values?: Value[]`（当 `type = enum` 时使用）
- `validation?: Validation`（约束规则集合）
- `properties?: Properties`（键值属性集合）

创建与写回逻辑：

- 通过 `moddle.create(`${prefix}:FormField`, { id, type, label })` 创建
- 根据 UI 输入可选填充 `defaultValue/datePattern`
- 最终写入 `formData.fields` 并通过 `updateElementExtensions()` 写回

对应代码见 [ElementForm.vue](../package/penal/form/ElementForm.vue#L295-L364)。

### 4.3 枚举值：Value（用于 enum）

类型：`${prefix}:Value`（如 `camunda:Value`）

- `id: string`
- `name: string`

在 `FormField.type = "enum"` 时，写入 `FormField.values`。

### 4.4 校验对象：Validation / Constraint

Validation 类型：`${prefix}:Validation`（如 `camunda:Validation`）

- `constraints: Constraint[]`

Constraint 类型：`${prefix}:Constraint`（如 `camunda:Constraint`）

- `name: string`
- `config: string`

写入位置：`FormField.validation.constraints`

### 4.5 属性对象：Properties / Property

Properties 类型：`${prefix}:Properties`（如 `camunda:Properties`）

- `values: Property[]`

Property 类型：`${prefix}:Property`（如 `camunda:Property`）

- `id: string`
- `value: string`

写入位置：`FormField.properties.values`

### 4.6 推荐的数据视图（便于前端对接）

默认面板内部维护两类数据：

- BPMN 可序列化对象（moddle 实例）：用于最终写回 XML（例如 `this.formData`、`Field`）
- UI 编辑用纯对象副本：用于 table/drawer/dialog 编辑（例如 `this.fieldList`、`this.formFieldForm`）

如果你自定义实现，建议也遵循该分层，避免将 UI 的临时状态直接写入 moddle 对象导致序列化异常。

## 5. 自定义实现时的读写建议

### 5.1 获取当前元素与建模器能力

属性面板初始化时，会把 bpmn-js 关键服务挂到 `window.bpmnInstances`：

- `window.bpmnInstances.bpmnElement`：当前选中元素
- `window.bpmnInstances.modeling`：用于写回属性（updateProperties / updateModdleProperties）
- `window.bpmnInstances.moddle`：用于创建扩展元素（moddle.create）

初始化代码见 [PropertiesPanel.vue](../package/penal/PropertiesPanel.vue#L142-L161)。

### 5.2 写入 formKey（表单标识）

推荐写法（与默认面板一致）：

```js
const element = window.bpmnInstances.bpmnElement;
window.bpmnInstances.modeling.updateProperties(element, { formKey: "FORM_001" });
```

### 5.3 写入扩展元素（FormData/FormField 等）

要点：

- 更新 `bpmn:extensionElements` 时需要保留其它扩展元素（例如监听器等）
- 默认实现会把“非 FormData 的扩展元素”保存在 `otherExtensions`，再拼接回去

参考实现：[ElementForm.vue](../package/penal/form/ElementForm.vue#L355-L364)。

## 6. 常见对接场景

### 6.1 只对接外部表单系统（只存表单 ID）

如果你的表单系统只需要在流程节点上绑定一个表单 ID：

- 使用 `formKey` 即可
- 不必写入 `FormData.fields`

### 6.2 需要字段级元数据（字段、校验、枚举、属性）

如果你希望在设计阶段配置字段元数据：

- 使用 `${prefix}:FormData` + `${prefix}:FormField` 结构写入到 `extensionElements`
- 运行时由流程引擎（或你自己的解析逻辑）从 BPMN XML 中读取扩展信息
